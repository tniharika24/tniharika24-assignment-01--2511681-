## RDBMS vs NoSQL

### Structure
RDBMS uses tables with rows and columns.
NoSQL uses flexible JSON-like documents.

### Schema
RDBMS has fixed schema.
NoSQL has dynamic schema.

### Relationships
RDBMS uses joins and foreign keys.
NoSQL uses nested data (embedding).

### Performance
RDBMS is better for structured queries.
NoSQL is faster for large-scale and flexible data.

### Use Case
RDBMS is used in banking systems.
NoSQL is used in real-time apps and big data systems.
